Multi-module Gradle Project Example
=====================================

This project shows a multi-module project composed of a root build file and two sub-modules: one containing the main web application (`my-webapp`) and the other a library (`my-app`) required by the web application as a dependency.

Compiling the project

    ./gradlew compileJava

Launching the tests

    ./gradlew test

Create the distributable version of all modules (JAR/WAR)

    ./gradlew build

The packaged version of `my-app` will be written to *my-app/build/libs* and can be run by adding the generated JAR to the classpath and specifying the main class

    java -cp my-app/build/libs/my-app-1.0-SNAPSHOT.jar com.mycompany.app.App

To delete and clean all the artifacts generated by previous builds, run the command

    ./gradlew clean

and check that the *build* directories in each module are removed.
